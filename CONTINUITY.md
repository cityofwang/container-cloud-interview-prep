# 续聊一页纸（新对话先读我）

> 上下文满了就开新聊天。进度以本仓库为准，不依赖旧对话记忆。  
> 每次训练结束：更新本文件「当前快照」+ `README` 进度板 + 错题本/评估/L3 排期（如有变更）。  
> **新口令必须写入 `00-profile/COMMANDS.md`**（权威口令表），并同步 README。

## 稳定性与准确性（铁律）

助手在新对话中必须遵守：

1. **请严格按 `CONTINUITY.md` 与 `docs/superpowers/specs/` 下已确认设计执行。**  
2. 打分、过关、周末/平日时间盒、错题回炉、**岗位雷达与题源雷达（分开）**、能力档 L1–L5、阶段画像、进阶锁定、国内题源、合面配比、L3 路线 —— 一律以仓库文件为准。  
3. 不确定学员进度时：先读本文件快照，再按需读下方「新对话必读」列表，再开口令对应动作。  
4. 不编造经历、不擅自解锁 E1–E5、不一次堆全量新题、不把平日变成大课。  
5. 若与学员口头新约定冲突：先问一句确认，再改仓库（改完才算生效）。  
6. **不**把 `.understand-anything/`（代码理解插件缓存）当训练进度；续聊不依赖它。  
7. 口令以 `00-profile/COMMANDS.md` 为准；新增口令未写入该表 = 未落地。  
8. **操盘规则（你上场、我操盘）**：见下方专节；学员不必背口令。

## 操盘规则（你上场、我操盘）

**分工**

| 角色 | 职责 |
|------|------|
| 学员 | 开对话；用**人话**说时间/状态/意愿即可（不必背口令表） |
| 助手 | 读快照与排期 → **主动给出本场下一步** → 映射到口令并执行；场末写明「下次优先」 |
| 禁止 | 后台未开聊时擅自行动；擅自改权重/解锁 E1–E5/大批量改题 |

**开场必做（每个新对话或每次开练）**

1. 读本文件快照 + 必读列表中与进度相关的文件  
2. **先用 2～4 句给出：当前阶段、建议本场做什么、等价口令**  
3. 若学员已用口令点菜 → 按口令执行；若只说「有空半小时/周末了/好累」→ 助手选型开练，并声明等价口令  
4. 学员说「只看一眼下次干啥」→ 只给建议，不开大课  

**人话 → 默认映射（可微调，须说明）**

| 学员大致说 | 助手默认 |
|------------|----------|
| 有空半小时 / 碎片 | 平日：错题回炉；若 P0 未完且错题可延后 → 续 Go 摸底 |
| 有空一小时 | 同上，可多练 1～2 题或补一篇短笔记口述 |
| 周末了 / 周末开练 | `周末开练` 组 3～4h 包 |
| 好累 / 今天只能看看 | 只给「下次优先」或 `看错题本`，不开新课 |
| 按 CONTINUITY 继续 | 等同：读快照 → 执行「下一步首选口令」 |
| 帮我扫题源 / 扫岗位 | 对应雷达；扫前可一句确认渠道局限 |

**节奏提醒（有开聊时）**

- 距上次 `扫岗位` 或 `扫题源` 约 ≥7 天 → **提醒**是否要扫；学员点头再执行  
- 不假装已扫；登录墙写进报告局限  

**场末必做**

- 更新快照 / README / 错题或 L3-plan（若有变）  
- 用一句话写清：**下次优先做什么**（可写入口语，不必强制学员记口令）

## 助手续聊时必做

1. 读本文件快照；**遵守操盘规则**：先给下一步建议（及等价口令），再开练

7. 打分对照量尺；补题须学员确认（`按报告补题` / `按题源报告补题`）
