## 1. 目标

1. 为 L1–L5（及 Offer 指向）落地**可阅读的能力画像**，不只保留分数门槛。
2. 题源与岗位雷达**拆开**：`扫岗位` = JD；`扫题源` = 牛客 / GitHub·个人笔记 / 公开面经等。
3. 全部对话口令集中登记；**以后每增一口令必须更新口令表**。

## 2. 阶段能力画像

路径：`00-profile/STAGE-PORTRAITS.md`

每档至少包含：

- 一句话定位（对标国内什么轮次/职级感）
- 八维大致分数带或关键维要求
- 能讲清的故事/生产深度
- Go / 编码要求
- 尚未要求什么（避免超前焦虑）

L3 为当前训练主目标；L4/L5/Offer 写清指向，细排期仍锁到达标后。

## 3. 题源双轨（方案 B）

| 口令 | 产出 | 渠道 |
|------|------|------|
| `扫岗位` | `review/job-market/YYYY-MM-DD-scan.md` | Boss/官网/公开 JD 等（沿用既有边界） |
| `扫题源` | `review/question-sources/YYYY-MM-DD.md` | 牛客面经、GitHub/优质个人笔记、公开中文面经汇总 |
| `看岗位雷达` / `看题源雷达` | 打开最近一次对应报告 | — |

题源报告固定结构：来源与局限 → 高频考点 → 与现图谱/题库缺口 → 补题建议 → 与阶段画像对齐说明。  
补题须学员确认（`按题源报告补题` / `按报告补题`）后再写入题包。

禁止：编造「某厂原题」；要求学员提供招聘/社区账号密码。
