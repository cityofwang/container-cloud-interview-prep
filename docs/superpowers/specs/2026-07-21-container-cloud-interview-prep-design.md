## 3. 各面次范围

### 一面｜基础过线（45–60 分钟）

- Docker 与容器隔离（namespace/cgroup 入门）
- Pod/Deployment/Service/Ingress/ConfigMap/Secret
- requests/limits、QoS 基础、探针、滚动更新/回滚、常见事件
- 监控告警链路
- 挂钩：日常排障、监控告警、事件驱动逻辑

### 二面｜生产深挖（60–90 分钟，主战场）

- CrashLoop / OOM / 节点 NotReady 等排障路径
- cgroup 与资源隔离（强项深挖）
- 在离线混部（场景、干扰、隔离、观测）
- CSI 部署流程与常见坑
- Service/CNI 现象级理解（A 向可再追一层）
- 挂钩：混部、cgroup、CSI、事件逻辑 → 1–2 个可深挖故事

### 三面｜架构与判断（约 60 分钟）

- 混部/资源方案边界与取舍
- 告警可行动性、变更/回滚/容量、故障复盘（STAR）
- 轻量系统设计（如 CSI/监控插件发布与灰度）
- 控制面/网络深度：知道边界 + 学习路径，不装懂

### 四面｜综合与软性（30–45 分钟）

- 项目交叉深挖、协作/值班压力
- 动机与成长表达（6 年偏运维如何讲清）
- HR：离职原因、期望、稳定性口径

### 旁线｜过线编码（贯穿全程）

- Shell/Python 排障/自动化小脚本
- 极简算法：数组 / 哈希 / 字符串
- 目标：过线，不按算法岗刷题
- 不挡面次推进；累计 ≥3 次小练习后再进三面

## 4. 单次训练流程

建议时长 60–90 分钟：

1. 热身 5 分钟：本轮目标与题包范围
2. 开卷题库 20–30 分钟：参考答法要点；标注「会 / 半会 / 不会」
3. 闭卷模拟 25–40 分钟：连续追问，按面试节奏作答
4. 复盘 15–20 分钟：评级、改写口述结构、写入 `mocks/`
5. 可选编码 15–20 分钟：记入 `05-coding-line/`

对话口令：

- `开始一面题库` / `开始模拟` / `复盘` / `补洞` / `编码练一题` / `过关检查`

## 5. 打分与过关

每题 1–5 分：

| 分 | 含义 |
|----|------|
| 5 | 结构清晰，有原理 + 生产例子，追问扛得住 |
| 4 | 主线正确，例子略浅或偶有含糊 |
| 3 | 概念对，但说不清为什么/怎么排 |
| 2 | 只能背关键词，经不起追问 |
| 1 | 不会或明显错误 |

每场模拟另评：技术深度 / 表达结构 / 与岗位匹配（各 1–5）。

进入下一「面」的条件：

- 本面「会 + 半会」≥ 80%，且「不会」已有口述补稿
- 最近一次模拟技术深度 ≥ 3.5，且无硬伤级错误
- 本面至少 1 个完整经历故事（情境→动作→结果→反思）
- 编码旁线不挡过关；进三面前累计 ≥3 次编码小练习

未过关：只补缺口 + 短模拟（20–30 分钟），不整面重开。

节奏建议：一面 1–2 周期；二面 2–3 周期；三面 1–2；四面 1 + HR 口径。
