## 1. 目标与范围

按既有画像与策略，培养可投递国内 **大厂/云厂商容器云 · 基础架构 · 混部** 方向的候选人，并具备 **云原生 Go 主 + 业务后端 Go 辅** 的合面表达与过线编码能力。

招聘要求来源：

- 基线：学员画像 + 八维评估 + 面次题库
- 动态：`扫岗位` → `review/job-market/YYYY-MM-DD-scan.md` → 经确认后补题/校准评估

### 1.2 长期目标分层（本 spec 只细做第一层）

| 层 | 含义 | 本 spec |

## 2. L3 达标定义

沿用 `assessment-and-unlock-gates`（不因本 spec 降低门槛）：

同时满足：

- 综合加权 ≥ **3.5**
- 正确率 ≥ **70%**
- 错题未巩固 ≤ **5**
- **C3 或 G2** 至少一维 ≥ **3.5**
- 至少完成 **1 次合面短模**，容器侧与 Go 侧均无高频硬伤（合面过关细则见 Go 双轨 / 面次设计）

含义：技术轮站得住；**不是**谈薪就绪（L5）或高薪定价主依据（L4）。

## 3. 时间假设与动态达标

### 3.1 默认时间盒

- 平日：≤30 分钟，默认错题回炉 / 碎片摸底续练
- 周末：一次 `周末开练`，约 3～4 小时
- 口令不变：见根目录 `README.md` / `CONTINUITY.md`

### 3.2 四阶段（L3）

| 阶段 | 内容 | 粗算耗时 | 出口标准 |
|------|------|----------|----------|
| **P0 摸底收口** | 续 Go 摸底 GO-D-07→08→09/10；定稿 `RESULT.md`；错题入库 | 1～2 个周末块，或 3～5 次 30min | RESULT 定稿；建议小节明确（当前倾向「两者」先 basics） |
| **P1 双轨打底** | `01-basics` + 并行 `开始一面题库`；薄册启动 OS 隔离/cgroup、网络到 Service/CNI 现象级 | 3～5 周 | basics 会+半会 ≥80% 且不会项有口述补稿；一面开卷比例够约短模 |
| **P2 并发 + 二面预习** | `02-concurrency` + 排障路径；混部/cgroup/CSI 故事口述；编码过线 1～2 题 | 3～5 周 | G2 抬升；错题未巩固趋向 ≤8 |
| **P3 合面冲 L3** | 合面短模 1～2 次 + 错题回炉；按雷达微调题感 | 2～4 周 | 达到 §2 L3 量尺 |

### 3.3 预计达标窗口（初始）

- 假设：每周 1 次周末包 + 平日错题基本执行
- **初始粗估：8～16 周**（自 2026-07-23 起 → 约 **2026-09-22～2026-11-12**）
- 当前快照偏弱（G1/G2 暂 2、错题未巩固 9、摸底未完）→ 默认按窗口 **偏后半** 沟通；执行质量高则可收至 8～10 周

### 3.4 动态改期规则（必须执行）

每次摸底定稿 / 错题回炉 / 合面 / `能力评估` 后，更新排期文件中的：

- 当前阶段（P0–P3）
- 预计达标周（区间上下界可各调 ±1～2 周）
- 阻塞项（未巩固错题数、硬伤考点）

强制顺延：

- 正确率掉档（连续 2 次 &lt;60%）或未巩固错题 &gt;10 → 下一包「回炉周」，达标日 **+1～2 周**
- 某周无周末包且平日断练 → 按实际缺失周数顺延
- `扫岗位` 可改补题方向与评估权重（经确认），**不擅自降低** L3 正确率/加权主门槛

## 4. 知识图谱

### 4.1 A 主图谱（L3 必达）

| 域 | 对应维 | L3 深度 | 主要载体 |
|----|--------|---------|----------|
| 容器/K8s 基础运维 | C1 | 一面对象与机制说清 | `01-round1/` |
| 生产排障与可观测 | C2 | 排障路径模板 + 常见故障 | `02-round2/` 入口 |
| 混部/cgroup/CSI | C3 | 故事可深挖、追问不倒 | 故事卡 + 二面 |
| Go 语言基础 | G1 | error/%w、interface、slice、defer 等 | `06-golang/01-basics` |
| Go 并发 | G2 | 泄漏、channel、context、小编码 | `06-golang/02-concurrency` |
| 云原生 Go | G3 | 事件/List-Watch 直觉即可先过 L3 | `06-golang/03-cloudnative`（可后置） |
| 编码过线 | X1 | 合面前累计练习次数达标趋势 | `06-golang/05-coding` |
| 结构化表达 | X2 | 四段式/排障/STAR | `00-profile/ANSWERING.md` |

### 4.2 B 薄册（够用层）

| 册 | 收录 | 明确不收录 |
|----|------|------------|
| OS | 进程/线程直觉、Linux namespace、cgroup、CPU/内存现象与容器对应 | 内核源码级调度器细节 |
| 网络 | TCP/DNS 基础、连通性排查、Service/CNI **现象与排查** | 自研 CNI、深度内核网络 |
| 算法 | 数组/哈希/字符串、超时取消类并发小题 | 竞赛图论/DP 题海 |
| 数据库 | **默认不单开**；若连续 ≥3 次雷达高频再提议观察维 | 系统学完 SQL 优化/事务课 |

### 5.2 单篇结构（统一）

1. **一句话结论**（面试背板）

- 学员口令建议：`补笔记 <知识点>` / 训练中教练判定「不会」时直接落篇
