## 0. 扫描方法说明（你问的「怎么做」）

**不是**整站全局爬虫备份题库。

| 步骤 | 做法 |
|------|------|
| 1. 关键词 | 对齐画像：容器云 / K8s / 混部·CSI / 云原生 / Go·GMP·channel·context·client-go |
| 2. 渠道抽样 | 牛客面经/话题、GitHub 面试笔记、公开中文博客（本轮公开可搜到的） |
| 3. 提炼 | 只记**高频考点与题感**，不整篇抄题、不声称「某厂原题」 |
| 4. 落仓 | 本报告；**补题须你确认**后再写入 `01-round*` / `06-golang` / `notes` |
| 5. 不备份 | 不把牛客/博客全文当私有题库镜像；保留链接 + 考点摘要即可复访 |

对比思维：岗（`扫岗位`）看「要什么人」；题（`扫题源`）看「怎么考」——分开、可交叉校准。

## 2. 高频考点

| 考点 | 出现感 | 偏 | 备注 |
|------|--------|----|------|
| 控制面组件 + Pod 创建/List-Watch | 高 | 容器 | 一面～二面；与你「事件」经历可挂钩 |
| Deployment 滚动/回滚/Ready 判断 | 高 | 容器 | 对比：暂停滚动 vs Istio 灰度（面经常见选型） |
| Service / kube-proxy（iptables vs ipvs）/ CNI 现象 | 高 | 容器 | 你弱项：现象级即可冲 L3，源码级偏 L4 |
| 调度：过滤打分、亲和/污点 | 中高 | 容器 | 混部岗加分 |
| CSI / PV·PVC | 中 | 容器 | **你强项**，应对齐深挖而非只背概念 |
| 排障：OOM、CrashLoop、NotReady | 高 | 容器 | 二面主战场 |
| Operator / Informer / reconcile 幂等 | 中高 | 综合 | 云原生 Go 岗高频；你 L3 要直觉，不强制大作 |
| Go：GMP 口述 | 高 | Go | concurrency 节必盖 |
| Go：channel / select / 关闭语义 | 高 | Go | 对应 GO-D-07 |
| Go：context 取消与传值边界 | 高 | Go | 对应 GO-D-08 |
| Go：泄漏 / pprof | 高 | Go | 已有笔记 GO-D-06 |
| Go：map 并发、interface nil、error 链 | 高 | Go | 摸底已覆盖 |
| 资源：cgroup / requests·limits / QoS | 中高 | 容器 | **你强项** |
| Mesh / 多集群 | 中（偏岗） | 综合 | 观察维 O4；L3 不主攻 |
