# 题源雷达（牛客 / GitHub·笔记 / 面经）

与 **岗位雷达**（`review/job-market/`）分离：这里只扫「考什么题、社区要求什么」，不扫 JD 薪资岗。

设计：`docs/superpowers/specs/2026-07-23-stage-portraits-and-question-sources-design.md`

## 口令

| 口令 | 作用 |
|------|------|
| `扫题源` | 立即扫描并写 `YYYY-MM-DD.md` |
| `看题源雷达` | 打开最近一次报告并讨论 |
| `按题源报告补题` | 学员确认后，把建议写入题包/待补清单 |

口令权威表：`00-profile/COMMANDS.md`（有新口令必须改那张表）。

## 渠道优先级

1. **牛客**等中文面试社区：容器云 / K8s / Go / 云原生面经与真题讨论（脱敏、不抄「原题」声称）
2. **GitHub / 个人笔记**：优质公开的 K8s·Go·混部 面试笔记、awesome 清单、系列博文
3. **公开中文面经汇总**（博客/知乎专栏等，注明局限）

少用：纯译国外题海、与国内栈无关的刷题列表。

## 报告路径

`YYYY-MM-DD.md`（见 `TEMPLATE.md`）

## 落库纪律

- 「补题建议」须学员确认后再改 `01-round*` / `06-golang/*` / `notes/`
- 每条新题遵守国内题感；标签可用：`牛客高频` / `笔记高频` / `面经变形`
- 不要求学员提供社区账号密码；登录墙写进「来源与局限」

## 扫描方法（固定）

1. **关键词抽样**（对齐画像），不是整站全局爬取  
2. 报告只存：**来源链接 + 高频考点 + 缺口 + 待确认补题建议**  
3. **不**把外站全文题库镜像进 git（版权与噪声）  
4. 你确认 `按题源报告补题` 后，才把**变形题**写入本仓题包  

最近报告：见目录下最新 `YYYY-MM-DD.md`。
